ServiceNow & Jira

Service Now/ JIRA Ticketing Systems & Tickets You Will Be Receiving

Public Plans Provider Services Hire Task for Public Plans E-Business/ Fulfillment Task for Public Plans E-Business Tickets

These tickets are for employees who need access to the portal. Within the ticket itself it would say whether they need access to Beta or Production and which state they are requesting access to. A Service-Now ticket will be assigned to group “Public Plans E-Business”. Once the ticket has been received, the ticket can be assigned to you or you can assign the ticket to yourself. To assign tickets to yourself right click on the ticket and click “Assign to me”.

For Internal Employees who need access to “HealthTrio Connect”

  1. A ticket will be sent through Service-Now by the employee’s manager

  2. Check to see which site the new employee needs access to (HealthTrio Connect Production or Beta for MA or RI) and whose access you need to mirror

  3. You will first look in both Beta and Production for MA and RI for an account for the user.

  4. If in the ticket it says the person was requesting access to Beta, but you find an account for them in Production, email them and verify which portal they need access to, and vice versa.

  5. If you find an account for them in the portal they requested access to you can confirm their account and then mirror their access to whomever they put on the ticket.

  6. If you don’t find an account for them email them stating that you did not find an account for them and in order to gain access to the portal they first have to register.

  • There are email stationaries available for MA and RI for both Beta and Production with links and instructions for the user

  • Put a work note in the Service-Now ticket explaining what you did.

  • Example: "User was not found Production Connect. I emailed user stating they would need to create an account in order to gain access to the portal."

  • If you had to email the user, change the status of the ticket from open to awaiting user info and save the ticket until they email you back with a confirmation.

  • Fulfillment and New Hire Tickets should only remain in ServiceNow for a maximum of three weeks. When you first see the ticket, check both the MA and RI portals to see if the person the ticket is for has a confirmed or pending account. If there is no account, email them using the “Connect Registration Internal” stationary found in the Connect mailbox. Make a note of this in the “work notes” section of the ticket after you have emailed them.

  • First attempt comment: An email has been sent to the user to register. This is the first attempt. On the third attempt the ticket will be closed.

  • If they have not reached out within a week of the first email, resend the email and CC the manager listed in the ticket. This is the second time you are trying to contact them. Note this second attempt in the “work notes” section of the ticket.

  • Second attempt comment: An email has been sent to the user and their manager/supervisor. This is the second attempt. On the third attempt the ticket will be closed.

  • If they still have not contacted you within a week of the second attempt, email the user with the “Expired Ticket” stationary found in the Connect mailbox. Be sure to CC their manager as well. You can now close the ticket.

  • Expired ticket comment: Ticket closed due to lack of response.

  • If the user registers and responds with “Connect registration complete”, confirm the account, mirroring the person requested. Leave a work note stating that the user is all set and close the ticket.

    • For example, if an employee requests access to MA Prod but hasn’t yet registered.

    Confirming Internal Employee’s Accounts:

    After you confirm the user’s account, you want to let them know that their account has been confirmed by sending an email to them and update the Service-Now ticket

    On the Connect site you will use the following functionalities to approve pending user accounts:

    1. Access List Manager
    2. Applicant Confirmations
    3. System Admin

    These functionalities are located on the left hand column in the User Administration section. The order of which to use these are typically System Admin, Access List Manager, and then Applicant Confirmation

    System Admin: System Admin is a search function for all users on Connect. You can search either by the user’s last name, user ID or company name. You can see whether they are a provider or member, if they have admin access or are a regular user, what their login user ID is, etc. This function is very useful. You will also need this function to add or delete accounts, add new roles, or for finding the Entity List Name for the Access List of the user. You can also create new accounts using this function by pressing the Add User button. When a new user is created a PDF user agreement is generated with the user’s log in information.

    Access List Manager: You will use this function 1-THPP- to search for, modify, and approve the Access Lists. You can also search for an Access List by its name or TIN-Practice (tax ID.) The Access List holds the following: TIN, Hospital ID, Practice ID, and Provider ID. Every provider account is linked to their company’s Access List. If the Access List lacks the proper information, the provider will be unable to properly view or create: claims, EOPs, authorizations, or affiliate providers. You can only add one TIN, Hospital ID, Practice ID, or Provider ID at a time to an Access Container. Open a new or preexisting Access Container and select the options from the drop down list on the New Access List Core Physicians LLC. Use the search function to find the matching TIN, Hospital ID, Practice ID, and Provider ID (don’t leave spaces).Once you select the right Valid Value List the page will reset. After it resets click Return to Access List Manager in the center. You are also able to delete single Entity Groupings, but be careful because you could ruin the provider’s access. If the Access List Name field lacks 1-THPP- , you can confirm the list by inserting 1-THPP- into the field at the beginning of the company and pressing the submit button(don’t leave any spaces). You can only approve the Access list if it has matching: TIN, Hospital ID, Practice ID, or Provider ID. If the Access List is missing any of these, you must not confirm it. Make sure that there are not any preexisting "1-THPP-" or "1-NH-" confirmed Access Lists of the same name and information before you confirm a new Access List. You want to avoid creating duplicates. After you have added all the proper information press the select button at the bottom to save any and all changes. Certain changes cannot be undone so be careful when editing.

    Applicant Confirmations: You will use this function to search for pending accounts. Make sure that the User field is set to Last Name and the Portal field is set to Provider. Type the prospective user’s name in the blank name field (you do not need to type the whole name). Make sure you have Starts With selected. Find the pending account and press select. To approve the pending account you need to confirm two things: the selected Role matches what is on the user agreement and that the Access List is confirmed with 1-THPP- —if the access list name is not confirmed you cannot approve the account. If this is a new Access List due to this being the first person from the practice to register, such as in the case of an owner who recently became contracted, update the name of the Access List to include 1-THPP- before the name of the practice. Make sure that the correct Officer has signed the user agreement before proceeding. Save your changes and confirm the account. Every confirmed Connect user account must have a 1-THPP- confirmation Access List. If the Access List does not have a 1-NH-, 1-THPP- or 1-THP- confirmation you will have to search for it in the Access List Manager. Once you find the right Access List copy and paste the ID number into the search function on the Applicant Confirmation Detail in the access list section. Then select the 1-THP- access lists that has the matching TIN on the user agreement. If there were multiple 1-THP- access lists with the same TIN select the one with the most users. Afterwards if the role does not match what’s on the User Agreement you can change it by clicking on the dropdown and then selecting the proper role. This must be done AFTER selecting the correct Access List otherwise it will be reset to the original role selected. You can then press the confirm button and get a confirmation message, and the user will be added to the system. You cannot confirm an employee account without first confirming the Main Office Manager’s administrative account.

    These three functions are all case sensitive, so make sure you don’t leave extra spaces, letters, or numbers when you make a search. When searching for the name of an Entity on Access List Manager it’s better to copy only some of the name and paste it into the search field. This way you are likely to get more search results.

    JIRA and Helpdesk

    You are responsible for provider verification and password resets. You should receive an email whenever a ticket on the NSC queue is created or updated. You will be assigned to tickets that require password resets.

    You may also be tasked to create and respond to tickets dealing with other Connect issues. Your supervisor or web support mentor will notify you when they need you to create or respond to JIRA tickets.

    If HealthTrio assigns us a member issue ticket forward the ticket to the Member Services department via email. Update the ticket stating that you have notified Member Services. Make sure to follow up with member services on the incident. You can then close the ticket once you know they have contacted the member.

    If you are assigned an incident which will cause a permanent change to the Connect system you will need to create a Change Management ticket. You will go to Helpdesk select the proper template and assign the ticket to both yourself and to the E-Business Queue. You will need to fill out the proper information, such as:

    • System: What system is being changed
    • Database/Environment: Production or Beta
    • Issue description: Describe the incident in detail
    • Implementation/Release Date: Date the change was implemented
    • Business Requestor: What part of the business requested the change
    • Initial HD (SR) #s: Prior tickets related to/affecting the change
    • Affected business areas: What part of the business was affected
    • Affected process: What part of the process was affected
    • Validated in Test by: Who tested it and checked that it works

    Password Resets

    User IDs and passwords to access the Connect Provider Portal are created and generated by HealthTrio Production support via the web portal. They are not created nor generated by the E-Business team via the Tufts Sponsor Configuration tools.
    Primary registrants are assigned non-temporary passwords during new user registration. Additional users added by the primary registrant during new user registration are assigned temporary passwords.

    • All temporary passwords have a 24-hour expiration time. This expiration time is set by HealthTrio Connect and not configurable in sponsor configuration tools. Non-temporary passwords that are self-designated by the user have a 90-day expiration time.
    • Password reset is a Production support feature by HealthTrio. The E-Business team cannot generate temporary or non-temporary passwords for a user.
    • During a password reset Production call, if HealthTrio is able to confirm all of the below with the user, HealthTrio will assign the user a temporary password
    • The only password reset ticket you would get in JIRA would be if the provider is the MOC. (Refer to HealthTrio Password Reset Process in the “For the Intern” folder)

    For Internal Employees:

    • Password resets for Internal employees will come into ServiceNow as Incident Tickets
    • Search for the user by last name in the portal instance they are requesting the reset for
    • Organize the list of results by company name and scroll down to Tufts Health Plan. This will make it easier to find the employee who needs their password reset.
    • Once you’ve found the user copy down their User ID from the User Maintenance page results. The User ID does not show up on user profile pages
    • Open JIRA in a new tab
    • Create a ticket requesting that the password be reset for the employee
    • Once you received the temporary password in the ticket reach out to the user telling them their password and put a comment under the Service Now ticket with the password in it.

    How to fill out the ticket

    • Only fill out the sections with the red asterisks
    • The first section with the red asterisk select EXT- THP from the drop down list
    • Next to summary put THP Internal Employee Password Reset
    • In the description you can copy and paste the following: Please reset the password for this THP employee.
    • In the Steps to Reproduce put the name of the employee and their User ID
    • If you wanted them to call the user you would need to leave their phone number
    • For the section that says priority it can stay routine since it’s only a password reset ticket
    • For the Business Impact section put None
    • Press create towards the bottom of your screen
    • When H30 updates the ticket with the temporary password email it to the user and let them know that it will expire in 24 hours

    For Providers

    • You should receive an email from the MOC approving the provider’s password reset request.
    • Once this is done you can create a JIRA ticket for the password reset
    • Once you received the password email or call provider with password

    Termination Ticket/Transfer out

    You are responsible for termination and transfer out tickets for the Connect Portal that come in to ServiceNow.

    When doing a termination/transfer out tickect make sure that the ticket is assigned to you or you assigned the ticket to yourelf by selecting your name in the “ Assigned to”drop down list.

    Select the request user last name, copy and paste it in the “System Admin” tab in both Beta and Production Connect. Make sure to check Beta MA & RI and Production MA & RI.

    Beta Connect:

    https://beta.healthtrioconnect.com/app/index.page

    Production Connect:

    https://thpprovider.healthtrioconnect.com/app/index.page?

    If you did not find the user anywhere in the Portal:

    • Comment in the “Work notes” tab indicating that the user was not found in either instance
    • Click on “Post” to post the comment
    • Scroll to the top of the ticket and click “Close Task” to close the ticket

    If the user was found in either Production or Beta (or both) :

    • Check the Termination date in the ticket.
    • Make a note of which instances the user was found in and post this information into the “work notes” section of the ticket.
    • If the termination date has passed remove the user’s access wherever you have found an account for them. Close the ticket.
    • If the termination date has not yet passed change the status of the ticket to “Work in Progress” and save and exit.
    • On the date of termination remove the account at the end of the day and close the ticket.

    Due Dates:

    • If you don’t find the user in Beta or Production you do not have to wait until the due date to close the ticket.
    • If you do find the user in either Beta or Production wait until the due date to remove their previous access and close the ticket

    Removing/Terminating User Accounts

    Search for the user by last name under System Admin

    On the User Maintenance screen, click on the user name you wish to delete

    On the User Role Maintenance section, check box for users and affiliated role(s) and click “Remove”

    On the “Are you sure you wish to remove this user?” Verification screen prompt:

    1: Enter reason (free-text):

    • Termination request/Terminated
    • Duplicate access
    2: Click the “Yes” Button or press the “Enter” key
